2026-6-10

项目架构设计

项目描述：

一个基于 RAG 架构的本地/云端 AI 知识库系统
技术栈：Vue3 + NestJS + Prisma + MySQL + LangChain + Qdrant

项目目标

LocalMind 是一个个人 AI 知识库系统，用户可以：

  • 上传 PDF / 文档
  • 构建知识库
  • 进行 AI 问答
  • 基于语义检索获取答案（RAG）

架构设计

系统采用 前后端分离 + AI增强架构：

表现层（Frontend）

技术栈：

  • Vue3
  • Axios
  • Pinia

职责：

  • 用户交互界面
  • 文件上传入口
  • 知识库管理UI
  • 聊天问答界面

应用层（Backend API）

技术栈：

  • NestJS
  • Prisma TypeORM
  • JWT

职责：

  • 用户认证（注册 / 登录）
  • 知识库管理
  • 文档上传处理
  • Chat对话接口
  • 调用 AI 服务链路

后端表单设计：

  1. User
  2. KnowledgeBase
  3. Document
  4. Chat

数据层（Database Layer）

MySQL（结构化数据）

存储内容：

  • 用户信息（User）
  • 知识库（KnowledgeBase）
  • 文档信息（Document）
  • 聊天记录（Chat）

Qdrant Chroma（向量数据库）

存储内容：

  • 文档切片向量
  • embedding结果
  • metadata（来源文档/知识库ID）

AI能力层（RAG核心）

LangChain

负责编排整个 AI 流程：

  1. 文档
  2. 切片
  3. 向量化
  4. 存储
  5. 检索
  6. Prompt
  7. LLM

Embedding模型

作用：

  • 将文本转换为向量
  • 支持语义相似度计算

大模型（LLM）

可选：

  • OpenAI API
  • Claude
  • Ollama（本地模型）

作用：

  • 根据上下文生成最终回答

开发中遇到的问题

Prisma 环境配置

Prisma 7 是官方重大升级，强制要求把数据库连接从 schema.prisma 移到 prisma.config.ts。 按官方文档配置后，始终报错、无法执行迁移、连不上数据库，完全无法开发**<prisma/schema.prisma**

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "mysql"
}
model User { ... }

prisma.config.ts

import "dotenv/config";
import { defineConfig, env } from "prisma/config";

export default defineConfig({
  schema: "prisma/schema.prisma",
  migrations: { path: "prisma/migrations" },
  datasources: {
    db: { url: env("DATABASE_URL") },
  },
});

env

DATABASE_URL=mysql://root:123456@localhost:3306/localmind?schema=public

执行命令报错

npx prisma migrate dev --config prisma.config.ts
Error: The datasource.url property is required in your Prisma config file when using prisma migrate dev.

无论怎么改：

  • 换 process.env.DATABASE_URL!
  • 换 env("DATABASE_URL")
  • 加 dotenv
  • 检查路径、空格、引号

全部无效，始终报同样错误，数据库完全连不上。

根本原因（Prisma7 重大 BUG / 兼容性问题）

Prisma 7 虽然官方强制推广 prisma.config.ts，但 CLI 命令（migrate/dev/generate）对 config.ts 兼容性极差、存在严重设计缺陷：

  1. migrate dev 命令不认 config.ts 里的 env () /process.env
  2. Prisma7 对 Windows + TS + MySQL 组合兼容性最差
  3. 报错信息误导：明明变量存在，却强制提示 url 缺失
  4. 官方文档与实际行为不一致
  5. config.ts 仅能被 generate 识别，migrate 完全不认

解决方案：彻底降级到 Prisma 6

Prisma 6 是稳定版、无 config.ts、直接 schema 写 env、Windows 完美兼容。

卸载 Prisma7

npm uninstall prisma @prisma/client
rm -rf node_modules/.prisma
rm -rf prisma/migrations
rm prisma.config.ts

安装 Prisma 6

npm install prisma@6.19.2 @prisma/client@6.19.2 --save-dev

改回 Prisma6 标准 schema

prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "mysql"
  url      = env("DATABASE_URL")
}

model User {
  id        Int      @id @default(autoincrement())
    username  String   @unique
    password  String
  createdAt DateTime @default(now())
    updatedAt DateTime @updatedAt
    chats     Chat[]
}

model KnowledgeBase {
  id        Int      @id @default(autoincrement())
    name      String
  documents Document[]
  createdAt DateTime @default(now())
    updatedAt DateTime @updatedAt
    }

model Document {
  id        Int      @id @default(autoincrement())
    name      String
  path      String
  type      String
    kbId      Int
  kb        KnowledgeBase @relation(fields: [kbId], references: [id])
    createdAt DateTime @default(now())
    }

model Chat {
  id        Int      @id @default(autoincrement())
    userId    Int
  user      User     @relation(fields: [userId], references: [id])
    question  String   @db.Text
    answer    String   @db.Text
    createdAt DateTime @default(now())
    }

.env 不变

env

DATABASE_URL=mysql://root:密码@localhost:3306/localmind?schema=public

执行命令

npx prisma migrate dev --name init
npx prisma generate

优化处理

改用 TypeORM

Prisma 缺点：

  1. Prisma 新版本 BUG 多，社区内容少，环境配置麻烦
  2. 占用大、 体积大、冷启动慢
  3. serverless 差
  4. 复杂查询啰嗦

TypeORM 优点：

  1. 官方包：@nestjs/typeorm，深度集成
  2. Nest 文档全是 TypeORM 示例，内容多，中文资料多

改 Qdrant 为 Chroma

Qdrant 缺点：

  1. 占用大
  2. 需要 Drocker 安装

Chroma 优点：

  1. 零配置、嵌入式、npm 直接装
  2. Nest/TS/LangChain 完美适配

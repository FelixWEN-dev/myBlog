一、常用工具库

VueUse

Vue3 组合式 API 工具函数集合，封装大量通用开发逻辑，简化代码、提升开发效率。还封装了各种浏览器 API、状态管理工具、时间处理函数等常用功能。

  • 模块化与按需加载：支持按需引入，优化性能。
  • 高可扩展性：支持插件扩展，如 Router、Firebase 等。
  • 社区活跃：文档详尽，功能更新迅速，适配最新 Vue 版本。

VueUse 的使用场景与示例

在脚手架项目中，我们使用前需要安装 @vueuse/core：

npm i @vueuse/core

安装好依赖后，我们就可以从 @vueuse/core 中引入相应的方法了：

<script setup>
import { useLocalStorage, useMouse, usePreferredDark } from "@vueuse/core";
// .....
</script>
状态管理

VueUse 提供了多种状态管理工具，可以轻松实现数据的响应式绑定与持久化。

1. useLocalStorage

将数据绑定到 localStorage，实现持久化和响应式。

<script setup>
import { useLocalStorage } from "@vueuse/core";
import { watch } from "vue";

// 使用 localStorage 存储用户主题偏好，默认为 "light"
const theme = useLocalStorage("theme", "light");

// 更新值时，localStorage 会自动更新
theme.value = "dark";
// localStorage["theme"] = "dark"

// 监听主题变化，响应式更新
watch(theme, (newValue) => {
  console.log(`主题切换为：${newValue}`);
});
</script>

2. useSessionStorage

与 useLocalStorage 类似，但数据保存在会话中，关闭浏览器后失效。

<script setup>
import { useSessionStorage } from "@vueuse/core";

// 会话级存储，页面关闭后失效
const sessionData = useSessionStorage("session", { user: "快乐就是哈哈哈" });
sessionData.value.user = "你好"; // 更新值
</script>
浏览器 API

VueUse 提供了多种封装良好的浏览器 API，便于操作。

1. useWindowSize

动态监听浏览器窗口大小。

<script setup>
import { useWindowSize } from "@vueuse/core";
import { watchEffect } from "vue";

const { width, height } = useWindowSize();
watchEffect(() => {
  console.log(`窗口大小：${width.value} x ${height.value}`);
});
</script>

2. useClipboard

用于读取和操作剪贴板内容。

<script setup>
import { useClipboard } from "@vueuse/core";

const { text, copy, isSupported } = useClipboard();

// 检查是否支持剪贴板功能
if (isSupported) {
  copy("VueUse 是强大的工具集！"); // 复制文本
  console.log("已复制到剪贴板");
}
</script>

3. useFullscreen

控制元素全屏显示。

<script setup>
import { ref } from "vue";
import { useFullscreen } from "@vueuse/core";

const el = ref(null);
const { isFullscreen, enter, exit, toggle } = useFullscreen(el);
</script>

<template>
  <div ref="el">
    <p>全屏内容区域</p>

    <button @click="toggle">
      {{ isFullscreen ? "退出全屏" : "进入全屏" }}
    </button>
  </div>
</template>
DOM 操作

通过 VueUse 操作 DOM 元素更加简单高效。

1. useElementSize

监听 HTML 元素大小。

<script setup>
import { ref } from "vue";
import { useElementSize } from "@vueuse/core";
import { watchEffect } from "vue";

const elementRef = ref(null);
const { width, height } = useElementSize(elementRef);

watchEffect(() => {
  console.log(`元素宽度：${width.value}px，高度：${height.value}px`);
});
</script>

<template>
  <div ref="elementRef">被监听的元素</div>
</template>

2. useDraggable

让元素支持拖拽功能。

<script setup>
import { ref } from "vue";
import { useDraggable } from "@vueuse/core";

const elementRef = ref(null);
useDraggable(elementRef); // 让该元素可拖拽
</script>

<template>
  <div ref="elementRef" style="position: fixed; cursor: move;">拖拽我</div>
</template>
时间管理

时间相关的 API 提供了实时功能支持。

1. useTimestamp

实时获取时间戳。

<script setup>
import { useTimestamp } from "@vueuse/core";
import { watchEffect } from "vue";

const timestamp = useTimestamp();
watchEffect(() => {
  console.log(`当前时间戳：${timestamp.value}`);
});
</script>

2. useNow

获取当前时间。

<script setup>
import { useNow } from "@vueuse/core";
import { watchEffect } from "vue";

const now = useNow();
watchEffect(() => {
  console.log(`当前时间：${now.value.toLocaleString()}`);
});
</script>

3. useDateFormat

格式化日期时间。

<script setup>
import { useNow, useDateFormat } from "@vueuse/core";

const now = useNow();
const formatted = useDateFormat(now, "YYYY-MM-DD HH:mm:ss");
</script>

<template>
  <p>当前时间：{{ formatted }}</p>
</template>

WangEditor

轻量富文本编辑器，适配 Vue 项目，用于后台系统文本内容编辑。

安装依赖

使用 npm 或 yarn 安装以下依赖：

npm install @wangeditor/editor-for-vue @wangeditor/editor

基本使用示例

<template>
  <div style="border: 1px solid #ccc">
    <!-- 工具栏 -->
    <Toolbar
      :editor="editorRef"
      :defaultConfig="toolbarConfig"
      mode="default"
      style="border-bottom: 1px solid #ccc"
    />
    <!-- 编辑器 -->
    <Editor
      v-model="valueHtml"
      :defaultConfig="editorConfig"
      mode="default"
      style="height: 500px; overflow-y: hidden"
      @onCreated="handleCreated"
    />
  </div>
</template>

<script setup>
import { ref, shallowRef, onBeforeUnmount } from "vue";
import { Editor, Toolbar } from "@wangeditor/editor-for-vue";

// 编辑器实例，必须用 shallowRef
const editorRef = shallowRef();

// 内容 HTML
const valueHtml = ref("<p>hello</p>");

// 工具栏配置
const toolbarConfig = {};

// 编辑器配置
const editorConfig = {
  placeholder: "请输入内容...",
  MENU_CONF: {
    uploadImage: {
      server: "/api/upload",
      fieldName: "file",
    },
  },
};

// 组件销毁时，也及时销毁编辑器
onBeforeUnmount(() => {
  const editor = editorRef.value;
  if (editor == null) return;
  editor.destroy();
});

const handleCreated = (editor) => {
  editorRef.value = editor;
};
</script>

nprogress

页面顶部进度条插件，用于路由切换、页面加载，优化页面加载体验。

安装

npm install nprogress

在 Vue Router 中使用

// router/index.js
import { createRouter, createWebHistory } from 'vue-router'
import NProgress from 'nprogress'
import 'nprogress/nprogress.css'

const router = createRouter({
  history: createWebHistory(),
  routes: [...],
})

// 路由开始跳转时显示进度条
router.beforeEach((to, from, next) => {
  NProgress.start()
  next()
})

// 路由跳转完成后关闭进度条
router.afterEach(() => {
  NProgress.done()
})

// 路由跳转出错时关闭进度条
router.onError(() => {
  NProgress.done()
})

export default router

自定义样式

/* 修改进度条颜色 */
#nprogress .bar {
  background: #409eff !important;
  height: 3px !important;
}

#nprogress .peg {
  box-shadow:
    0 0 10px #409eff,
    0 0 5px #409eff !important;
}

husky

Git 钩子工具，拦截代码提交、推送，校验代码规范，统一团队代码风格。

安装与配置

# 安装 husky
npm install husky --save-dev

# 初始化 husky（v9 版本）
npx husky init

配置提交前钩子（pre-commit）

# 创建 pre-commit 钩子，在提交前自动运行 lint
npx husky add .husky/pre-commit "npx lint-staged"

配合 lint-staged 使用

npm install lint-staged --save-dev

在 package.json 中配置：

{
  "lint-staged": {
    "*.{js,vue,ts}": ["eslint --fix", "prettier --write"]
  }
}

配置提交信息规范（commit-msg）

npx husky add .husky/commit-msg 'npx --no-install commitlint --edit "$1"'

配合 @commitlint/config-conventional 使用，强制提交信息符合 Conventional Commits 规范。

jsdom

Node 端模拟浏览器 DOM 环境的工具，无需浏览器即可操作 DOM，多用于前端单元测试。

安装

npm install jsdom --save-dev

基本使用示例

const { JSDOM } = require("jsdom");

const dom = new JSDOM(
  `<!DOCTYPE html><html><body><div id="app"></div></body></html>`,
);

// 获取 window 和 document
global.window = dom.window;
global.document = dom.window.document;

// 操作 DOM
const app = document.getElementById("app");
app.innerHTML = "<h1>Hello JSDOM</h1>";
console.log(app.innerHTML); // <h1>Hello JSDOM</h1>

在 Vitest 中使用

// vitest.config.js
import { defineConfig } from "vitest/config";

export default defineConfig({
  test: {
    environment: "jsdom", // 使用 jsdom 模拟浏览器环境
  },
});
<!-- Counter.vue -->
<template>
  <div>
    <span>{{ count }}</span>

    <button @click="increment">+1</button>
  </div>
</template>

<script setup>
import { ref } from "vue";
const count = ref(0);
const increment = () => count.value++;
</script>
// Counter.spec.js
import { describe, it, expect } from "vitest";
import { mount } from "@vue/test-utils";
import Counter from "./Counter.vue";

describe("Counter", () => {
  it("点击按钮后计数增加", async () => {
    const wrapper = mount(Counter);
    const button = wrapper.find("button");
    await button.trigger("click");
    expect(wrapper.find("span").text()).toBe("1");
  });
});

二、Vue3 实战代码

1. 路由页面缓存

<template>
  <RouterView v-slot="{ Component }">
    <keep-alive :include="cachedViews">
      <component :is="Component" v-if="isCacheable" />
    </keep-alive>

    <component :is="Component" v-if="!isCacheable" />
  </RouterView>
</template>

作用：按需缓存路由页面，提升页面切换性能。keep-alive 会缓存组件实例，避免重复渲染和请求数据。

注意事项：

  • 被缓存的组件需要设置 name 选项，且与 include 中的名称一致。
  • 可以在路由元信息中配置 meta.keepAlive 来动态控制。

2. 后台侧边栏布局

<template>
  <el-container class="admin-layout">
    <el-aside width="220px" class="layout-sidebar">
      <AppSidebar :menus="staticMenus" :active-menu="activeMenu" />
    </el-aside>

    <el-container>
      <el-header class="layout-header">
        <AppHeader />
      </el-header>

      <el-main class="layout-main">
        <RouterView />
      </el-main>
    </el-container>
  </el-container>
</template>

作用：搭建后台管理系统侧边栏布局，传入菜单数据与激活菜单状态。

3. 侧边栏菜单渲染（递归组件）

<!-- SidebarMenuNode.vue -->
<template>
  <el-sub-menu v-if="hasChildren" :index="node.path">
    <template #title>
      <el-icon><component :is="node.meta?.icon" /></el-icon>

      <span>{{ node.meta?.title }}</span>
    </template>

    <SidebarMenuNode
      v-for="(child, index) in node.children"
      :key="child.path || `sub-${index}`"
      :node="child"
    />
  </el-sub-menu>

  <el-menu-item v-else :index="node.path" @click="handleClick">
    <el-icon><component :is="node.meta?.icon" /></el-icon>

    <template #title>{{ node.meta?.title }}</template>
  </el-menu-item>
</template>
<!-- AppSidebar.vue -->
<template>
  <el-menu
    background-color="#304156"
    text-color="#bfcbd9"
    active-text-color="#409eff"
    :default-active="activeMenu"
    @select="handleSelect"
    router
  >
    <SidebarMenuNode
      v-for="(item, index) in sidebarMenus"
      :key="item.path || `root-${index}`"
      :node="item"
    />
  </el-menu>
</template>

<script setup>
import SidebarMenuNode from "./SidebarMenuNode.vue";

defineProps({
  menus: {
    type: Array,
    default: () => [],
  },
  activeMenu: {
    type: String,
    default: "",
  },
});

const emit = defineEmits(["select"]);

const handleSelect = (index, indexPath) => {
  emit("select", index, indexPath);
};
</script>

作用：循环渲染多级侧边栏菜单，监听菜单选中事件。使用递归组件 SidebarMenuNode 实现无限层级菜单渲染。

三、Vue3 插槽类型及作用

  • 默认插槽：无命名插槽，适用于单一处内容插入。
  • 具名插槽：带名称插槽，用于组件多位置分发内容。
  • 作用域插槽：父组件可获取、使用子组件内部数据。
